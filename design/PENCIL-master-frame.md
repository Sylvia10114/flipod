# Pencil Agent 提示词 — Master Frame 布局骨架

## 目标

建立一个统一的页面布局骨架（Master Frame），所有 Player 相关画板必须基于这个骨架派生。解决当前各页面之间顶部间距、字幕区起始位置、底部控制栏不统一的问题。

屏幕尺寸：390 x 844（iPhone 14/15 logical）

---

## 布局分区定义（6 个固定区域）

将 844px 的垂直空间划分为 6 个**固定高度**的区域。无论页面处于什么状态（正常播放、点词弹窗、Loading、Re-rank），每个区域的 y 坐标和高度都不变。

```
┌─────────────────────────────┐  y=0
│  A. 状态栏 (Status Bar)      │  h=54
├─────────────────────────────┤  y=54
│  B. 顶部信息区 (Header)      │  h=130
│    菜单图标 / 标题 / 来源     │
│    hint 文字 / AI reason     │
├─────────────────────────────┤  y=184
│                             │
│  C. 字幕区 (Subtitle)        │  h=300
│    英文字幕 + 中文翻译        │
│    （垂直居中于此区域内）      │
│                             │
├─────────────────────────────┤  y=484
│  D. 侧边操作区参考线          │  （叠加在 C/E 之间）
│    ❤️ 🔖 固定在 right:12     │  y=484, h=80
├─────────────────────────────┤  y=564
│  E. 空白缓冲区               │  h=80
│    （字幕与控制栏之间的呼吸）   │
├─────────────────────────────┤  y=644
│  F. 底部控制区 (Controls)     │  h=200
│    进度条 / 播放按钮 /        │
│    状态栏（速度、clip 编号）   │
└─────────────────────────────┘  y=844
```

---

## 各区域详细规格

### A. 状态栏 — y:0, h:54

- 系统状态栏（时间、WiFi、电量），由系统渲染
- 不放任何自定义元素
- 所有画板此区域完全一致

### B. 顶部信息区 — y:54, h:130（底边 y=184）

这个区域容纳**所有**顶部元素。关键原则：**无论元素显示与否，区域高度不变，内容垂直居中于此区域。**

| 子元素 | 相对位置 | 说明 |
|--------|---------|------|
| 菜单图标 ☰ | 区域内 top:6, left:20 | 32x32，仅在正常播放态显示 |
| 帮助图标 ? | 区域内 top:6, right:20 | 32x32，Loading/Fallback 态隐藏 |
| Hint 文字 | 区域内 top:40, 居中 | 11px，状态相关文案 |
| Clip 标题 | 区域内 top:62, 居中 | 16px medium，--text-1 |
| 来源 + 标签 | 区域内 top:88, 居中 | 12px，podcast name · tag |
| AI Reason | 区域内 top:108, 居中 | 12px，--text-3，可选显示 |

**状态变化时的处理：**

- **正常播放态**：所有子元素显示，按上表排列
- **点词弹窗态**：菜单图标隐藏，但区域高度不变（空间保留）。标题和来源位置不动
- **Loading 态**：Hint 显示 "AI 正在为你排列内容..."，标题/来源/AI Reason 隐藏，但区域高度不变
- **Fallback 态**：AI Reason 和 ? 图标隐藏，其余不变

### C. 字幕区 — y:184, h:300（底边 y=484）

- 左右边距：32px（内容宽度 326px）
- 内容在区域内**垂直居中**
- 英文字幕：22px，行高 1.5，居中对齐
- 中文翻译：14px，行高 1.4，--text-3，居中对齐
- 字幕与翻译间距：16px
- **所有页面的字幕区起始 y 坐标 = 184，没有例外**

### D. 侧边操作区 — y:484, right:12, h:80

- 叠加层，不占据主布局流
- ❤️ 心形图标：20x20，y=484
- 🔖 书签图标：20x20，y=528
- 间距：24px
- 所有画板此位置固定，不随字幕长度变化

### E. 空白缓冲区 — y:564, h:80

- 纯留白，作为字幕区与控制区之间的视觉呼吸空间
- Word Popup 弹窗的锚点在此区域上方：`bottom:220px` 即 y≈624 处的底边
- 不放任何持久元素

### F. 底部控制区 — y:644, h:200（底边 y=844）

从上到下排列：

| 子元素 | 相对位置（区域内） | 说明 |
|--------|-----------------|------|
| 进度条行 | top:0 (y=644) | 时间 + 进度条 + 时长，左右边距 32px |
| 播放控制行 | top:50 (y=694) | ⏪ ▶️ ⏩，居中，gap 40 |
| 状态栏行 | top:110 (y=754) | 左：👁 + A + 1.0x ｜ 右：clip 编号 |
| 安全区留白 | top:140 → bottom | iPhone 底部安全区 |

## Pencil 操作步骤

### Step 1：创建 Master Frame Symbol

1. 新建画板 "MASTER — Player Frame"，390x844
2. 画 6 个矩形，分别对应 A-F 区域，颜色用低透明度填充（仅作参考线，不是最终视觉）：
   - A 区 (0,0,390,54)：`rgba(255,0,0,0.05)` 红色参考
   - B 区 (0,54,390,130)：`rgba(0,255,0,0.05)` 绿色参考
   - C 区 (0,184,390,300)：`rgba(0,0,255,0.05)` 蓝色参考
   - D 区 (350,484,40,80)：`rgba(255,255,0,0.08)` 黄色参考（侧边）
   - E 区 (0,564,390,80)：无色（纯留白）
   - F 区 (0,644,390,200)：`rgba(128,0,255,0.05)` 紫色参考
3. 锁定所有参考矩形，设为不可选中
4. 将整个画板转为 **Symbol**（或 Component），命名为 `MasterPlayerFrame`

### Step 2：基于 Master Frame 派生所有 Player 画板

以下画板全部从 `MasterPlayerFrame` 实例化（Insert Symbol），然后在实例上叠加各自的内容元素：

| 画板名 | 说明 | B 区特殊处理 | C 区特殊处理 |
|--------|------|-------------|-------------|
| s4 Feed 播放 | 正常播放态 | 全部元素显示 | 英文+中文字幕 |
| s5 点词 Popup | 点击单词态 | 菜单图标隐藏，其余不动 | 字幕 + Word Popup 叠加 |
| s6 Re-rank | 重排提示态 | Hint 高亮 40% 白 | 字幕不变 |
| s7 Loading | 加载态 | 仅 Hint 显示 | 三个 loading dots 居中 |
| s8 Fallback | 降级态 | 无 ? 图标和 AI Reason | 字幕正常 |
| s13-s18 Practice | 练习模式各步骤 | 标题改为练习相关 | 内容区按练习逻辑 |

### Step 3：对齐检查

完成所有画板后，逐一检查：

1. **B 区底边对齐**：所有画板的 Clip 标题 y 坐标 = 116（绝对坐标），允许 ±0 误差
2. **C 区顶边对齐**：所有画板的英文字幕首行 y 坐标在 184-280 范围内（垂直居中浮动）
3. **F 区顶边对齐**：所有画板的进度条 y 坐标 = 644，允许 ±0 误差
4. **侧边按钮对齐**：所有画板的 ❤️ 图标 y 坐标 = 484

## 注意事项

1. **区域高度是刚性的**——元素隐藏时区域不塌缩，只是内容不显示
2. **B 区是唯一跨状态变化的区域**——但它的高度锁死 130px，内部元素做 visibility 切换而非移除
3. **Word Popup 是浮动层**——不属于任何区域，固定在 `bottom:220px, left:50%`（大约 y=624 底边），叠加在 C+E 区之上
4. **Practice 模式**可能需要 C 区内部的子布局调整，但 C 区的起止坐标（184-484）必须和 Feed 播放保持一致
