# PRD: Feed + Practice 双模式架构

## 一句话

Feed 是刷播客，Practice 是练听力。两件事分开做，互不干扰。

---

## 问题

上一版 PRD 试图把学习藏进听的过程里，结果要么太隐（用户感知不到），要么一显就成了传统 app 那套插卡片。根本原因是把"听"和"练"混在一个场景里——这两件事的用户心态完全不同：

- 听：我想放松/获取信息，别烦我
- 练：我愿意花力气，给我结构化的指导

硬把练习塞进听的流程 = 在用户想放松的时候逼他做题。
硬把听的体验做成练习 = 在用户想学的时候什么都不教。

**所以分开。**

---

## 架构

```
┌──────────────────────────────────────────┐
│               Listen Leap                 │
│                                          │
│  ┌───────────┐  收藏   ┌──────────────┐  │
│  │   Feed    │ ──────> │  Practice    │  │
│  │  (泛听)   │         │  (精听)       │  │
│  │           │         │              │  │
│  │ 刷播客片段  │         │ 侧边栏入口     │  │
│  │ 发现内容   │         │ 收藏 + AI推荐  │  │
│  │ 收藏感兴趣的│         │              │  │
│  └───────────┘         └──────────────┘  │
│        │                      │          │
│        └──────────┬───────────┘          │
│                   ▼                      │
│           Vocab (词汇积累)                │
│           Feed点词 + Practice精听         │
│           自动沉淀，不需要单独入口           │
│                                          │
└──────────────────────────────────────────┘

用户操作闭环：
Feed刷→收藏→Practice精听→词汇沉淀→AI用词汇调整Feed推荐→...
```

### Feed 模式（现有的，播放器升级）

上下滑动刷播客片段，字幕跟随，点词看释义，中文翻译可遮挡。AI 排序推荐。

**不加任何教学元素。** 不加 recap，不加填空，不加闪卡。Feed 的指标是"用户愿意继续刷"，不是"用户学到了多少"。

Feed 不新增 Practice 入口。播放器本身做两项基础能力升级（见下方"播放器升级"章节）。

### Practice 模式（新增的核心功能）

入口只在侧边栏。用户不是"从 Feed 里点进 Practice"，而是主动进入一个独立的学习场景。

Practice 的内容来源 = **用户收藏过的 clip** + AI 推荐。这形成一个关键心智模型：**Flipod 里的每个操作都有用**——你在 Feed 里收藏一段内容，它不是存起来就完了，它会出现在 Practice 里等你精听。

这套流程来自被验证了几十年的听力教学方法论（精听法）。我们做了两个关键裁剪：砍掉跟读（那是口语训练，不是听力训练）和听写（手机打字体验差，太像考试）。剩下的 4 步是纯听力闭环——用户从头到尾只需要"听"和"点"，不需要开口、不需要打字，通勤和睡前都能做。

## 播放器升级（Feed 和 Practice 共享）

以下两项是播放器的基础能力升级，**不是教学功能**，在 Feed 和 Practice 中都生效。

### 单句跳转

- skip-back / skip-forward 改为跳到上一句 / 下一句（已有 word-level 时间戳和 segment 分句数据，技术直接支持）
- 进度条上用极细竖线（1px，`rgba(255,255,255,0.10)`）标出句子边界
- 当前句子对应的进度段用 accent 色，其他已播放段用更暗的颜色
- 底部状态栏显示句子指示器："第 3 / 8 句"

**为什么是基础能力**：听播客想重听某句话是所有用户的需求，不只是学习者。按句跳比按 5 秒跳精准得多。

### 惯用手模式

播放器支持左手/右手模式切换（设置里选择，默认右手）。

**右手模式（默认）：**
- Side Actions（心形、书签）在右侧
- 中文翻译遮挡的点击热区偏右（右手拇指自然覆盖区）
- 进度条拖动区域不变（全宽）

**左手模式：**
- Side Actions 镜像到左侧
- 中文翻译遮挡的点击热区偏左
- 其他布局水平镜像

**为什么做这个**：单手持机是主要使用场景（通勤、睡前）。左撇子用户用右手布局需要跨屏点击，体验明显差。实现成本低——CSS mirror 或 RTL-like 布局即可。

## Practice 流程（4 步，纯听力）

以下步骤作为连续的全屏体验，自动推进。整个过程用户只需要**听**和**点**，不需要开口、不需要打字。通勤、睡前、任何场景都能做。

### Step 1：盲听

- 播放整段 clip，**不显示任何字幕**
- 屏幕只有音频波形动画 + 播放控制
- 顶部提示："先听一遍，看能抓住多少"
- 播放结束后，底部出现两个选项：
  - "大部分听懂了" → 跳到 Step 4（复听）
  - "有些没听清" → 进入 Step 2（逐句精听）
- 用户自我评估，没有对错判断

**设计意图**：让用户先对自己的理解程度有个感知。听懂了的人不需要做精听，直接复听确认即可。

### Step 2：逐句精听

- 把 clip 拆成单句，一句一句播放
- 每句播放后暂停，屏幕显示：
  - 英文字幕（高亮当前句，其余灰暗）
  - 可点击的单词（点击弹释义 popup，跟 Feed 里一样）
  - 中文翻译默认隐藏，点击显示
- 用户可以：
  - 重听当前句（点击重播按钮）
  - 标记"这句没问题"（右滑 / 点 ✓，跳到下一句）
  - 标记"这句有难度"（左滑 / 点 ✗，这句会在 Step 3 出现）
- 底部进度条显示"第 3 / 8 句"

**设计意图**：精听的核心环节。逐句过，不懂的查，有针对性地标记难点。整个过程是"听→看→理解"，不需要输出。

### Step 3：难句闪卡

- 把 Step 2 中用户标记"有难度"的句子，以闪卡形式逐张呈现
- 每张闪卡：
  - 正面：播放该句音频 + 英文字幕（目标难词高亮）
  - 翻面（点击/下滑）：中文翻译 + 难词释义 + 音标
- 用户可以：
  - "搞懂了" → 这张卡消失
  - "还是不太清楚" → 这张卡排到最后再出现一次
- 没有标记难句的话，跳过这步

**设计意图**：只复习你不会的，不浪费时间在已经懂的句子上。闪卡在这里是合理的——用户主动进了练习模式，期望就是学东西。而且闪卡的核心交互仍然是"听"（先播音频），不是看图选义。

### Step 4：复听

- 完整播放整段 clip，附带英文字幕
- 跟 Feed 里的播放体验几乎一样，但：
  - 之前查过的词会有一个微标记（词下方的小点，表示"你练过这个词"）
  - Step 2 中标记过"有难度"的句子，进度条上对应位置有 accent 色标记
- 播放结束后，显示一个简单的总结：
  - "这段练完了 · 查了 4 个词 · 精听了 8 句"
  - 两个按钮："回到 Feed" / "再练一段"

**设计意图**：完成闭环。从"一开始听不太懂"到"现在完全听懂了"，这个对比本身就是最好的学习反馈。不需要分数，用户自己能感受到差异。

## 入口设计

### 唯一入口：侧边栏

Practice 只从侧边栏进入，Feed 播放器上不加任何练习入口。这是刻意的——Feed 和 Practice 是两个心智空间，入口也要物理隔开。

侧滑面板菜单新增一项："🎧 听力练习"，排在"📚 我的收藏"和"📝 词汇本"之间。

进入后的页面结构：

**顶部区域 — 收藏的 clip（优先展示）**
- 标题："你收藏的"，14px，`--text-2`
- 展示用户收藏过但未练习过的 clip，按收藏时间倒序
- 每个卡片：clip 标题 + 播客名 + 时长 + "未练习"标签
- 已练习过的收藏卡片灰度降低，标注"已练习 · 4.8 查了 3 个词"
- **如果没有收藏过任何 clip**：显示空状态——"在 Feed 里收藏感兴趣的片段，它们会出现在这里等你精听"

**下方区域 — AI 推荐**
- 标题："AI 推荐练习"，14px，`--text-2`
- 3-5 个 AI 推荐的 clip（基于用户水平、之前的词汇标记、未听过的内容）
- 每个卡片：标题 + 播客名 + 预估练习时间（如"约 4 分钟"）+ 难度标签
- 推荐理由（一行小字）："包含你上次查过的 cortisol" 或 "难度适合你现在的水平"

点击任意卡片 → 进入 Practice Step 1。

### 功能引导（Onboarding）

用户首次收藏一个 clip 时，触发一次性引导提示：

- 收藏动画完成后，底部浮现一行提示（toast 样式，2.5 秒后自动消失）：
- "已收藏 · 可以在侧边栏「听力练习」里精听这段"
- 带一个小箭头指向左上角菜单方向

用户首次打开侧边栏"听力练习"时，触发一次性引导：

- 页面顶部显示一段简短说明（可关闭）：
- "精听练习：对一段内容反复听、逐句听、搞懂每个词。比泛听累，但进步快。"
- 下方自动展示用户收藏的 clip（如果有的话），或 AI 推荐的 clip

### 心智模型：Flipod 里的每个操作都有用

这个设计要传达的信息是：

| 用户操作 | 产生的效果 |
|---------|-----------|
| 在 Feed 里刷 clip | AI 学习你的偏好，推荐更准 |
| 点词查释义 | 词进入词汇本，AI 在后续 clip 中自然复习 |
| 收藏一个 clip | clip 出现在"听力练习"里，等你精听 |
| 在 Practice 里标记难句 | 难句进入闪卡复习，词标记为"精听过" |

没有一个操作是白费的。用户会逐渐意识到：**我在这个 app 里的每一次交互，都在让它更懂我、让我学得更多。**

## 词汇沉淀

不需要单独设计"词汇复习模块"。词汇在两个场景中自然积累：

1. **Feed 里点词** → 词进入词汇本
2. **Practice 里查词/标记难句** → 词进入词汇本，且带有"练习过"标记

词汇本已有设计（Panel — Vocab），保持不变。唯一新增：

- 每个词条增加"来源"信息：来自哪个 clip，是在 Feed 浏览时查的还是 Practice 练习时遇到的
- Practice 中遇到的词标记为"精听过"，权重更高（因为用户花了更多注意力）
- AI 在排列 Feed 时，偏向推送包含用户"精听过"的词的新 clip，实现自然复习

## 不做的事

| 提案 | 为什么不做 |
|------|-----------|
| Feed 中插入任何学习卡片 | Feed 是消费场景，不是学习场景 |
| 跟读/shadowing | 这是口语训练，不是听力训练。而且不录音不评分等于没反馈，通勤/睡前也没法做 |
| 录音评分 | 技术复杂度高，用户隐私顾虑，而且评分带来焦虑 |
| 每日目标/打卡 | 制造愧疚感而非学习动力 |
| 听写（dictation） | 手机上打字体验差，而且听写对中国用户来说太像考试 |
| 全量闪卡复习（Anki模式） | 只复习用户标记过"有难度"的内容，不做全量 SRS |

---

## 为什么这个方案不同于传统 app

**跟 Duolingo 的区别**：Duolingo 的练习材料是为教学设计的假场景。我们的练习材料是用户刚在 Feed 里真实感兴趣的播客内容。你练的是你想听懂的东西，不是课本上的对话。

**跟百词斩的区别**：百词斩是"背词 → 找场景"。我们是"听到真实场景 → 对场景做深度练习 → 词汇自然沉淀"。词不是目标，理解内容是目标。

**跟所有英语 app 的区别**：没有任何 app 是"先让你刷真实播客，然后对你感兴趣的那段做精听训练"。这个 Feed → Practice 的流转是产品核心差异。

## 需要 Pencil 出的画板

### 播放器升级（2 个画板）
1. **Player — Sentence Nav**：进度条带句子分段标记，前进后退为句子跳转图标，底部句子指示器"第 3 / 8 句"
2. **Player — Left-hand Mode**：左手模式下 Side Actions 和交互热区镜像到左侧（与右手模式对比展示）

### Practice 入口（3 个画板）
3. **Panel — Practice（有收藏）**：侧边栏练习页面，顶部展示用户收藏的 clip（含"未练习"和"已练习"两种状态），下方 AI 推荐区
4. **Panel — Practice（空状态）**：无收藏时的空状态引导——"在 Feed 里收藏感兴趣的片段，它们会出现在这里"
5. **Feed — Bookmark Toast**：用户首次收藏时底部浮现的引导提示

### Practice 流程（5 个画板）
6. **Practice — Step 1 盲听**：纯音频波形 + 播放控制 + "大部分听懂了/有些没听清"两个选项
7. **Practice — Step 2 逐句精听**：单句字幕高亮 + 点词释义 + 左右滑标记难度 + 句子进度 "第 3 / 8 句"
8. **Practice — Step 3 难句闪卡（正面）**：播放音频 + 英文字幕，难词高亮
9. **Practice — Step 3 难句闪卡（背面）**：中文翻译 + 难词释义 + 音标 + "搞懂了/还是不太清楚"
10. **Practice — Step 4 复听 + 总结**：完整播放（带精听标记）+ 练完的总结数据 + "回到 Feed/再练一段"
