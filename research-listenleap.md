# ListenLeap 竞品调研报告

**调研日期**：2026-04-07

---

## 1. 产品概述

ListenLeap 是一个英语听力学习 App，核心理念是把**真实的播客和 YouTube 视频**变成可学习的材料。基于 Krashen 的"可理解输入"理论——通过接触真实内容习得语言，而不是教材练习。

- **开发商**：INSPIRED AI PTE. LTD.（新加坡，2023 年成立）
- **姊妹产品**：TalkMe AI（口语对话 AI 教练，100 万+用户）
- **融资**：2024 年 11 月天使轮 350 万美元（Seas Capital、锦秋基金、金沙江创投）
- **上线时间**：2024 年 10 月（iOS），2025 年 5 月（Android）
- **当前版本**：v1.7.1（iOS）
- **评分**：iOS 4.8/5（96 评分），Android 4.5/5（297 评论，1 万+下载）
- **预估月收入**：约 $77K（iOS，AppstoreSpy 估算）
- **核心市场**：台湾、香港、中国大陆、韩国、澳门

## 2. 核心功能

### 2.1 内容库
- 10,000+ 播客剧集，来源：BBC、TED、NPR、AEE、iTunes/Spotify 热门
- 20+ 主题分类（新闻、商业、犯罪、科技、历史、儿童、英音等）
- 每日更新
- 支持用户导入 YouTube 视频 URL，自动生成双语字幕
- v1.7.1 新增：导入本地音视频文件

### 2.2 播放界面（核心学习屏）
- **双语字幕**：英文 + 学习者母语（繁中、简中、韩、日、越），逐句滚动同步
- **变速播放**：0.5x ~ 2.0x
- **单句循环**：点击某句可反复听
- **全集循环**
- **点词查询**：长按任意单词 → 即时释义、发音、用法例句
- **AI 问答**：点击任意词/句 → AI 给出语法分析、文化背景、使用场景
- **三种显示模式**：日间模式、单句模式、夜间模式
- **可调字体大小**
- **睡眠定时器**

### 2.3 跟读练习（Shadowing）
- 用户录音模仿播客语音
- AI 实时评分：发音（pronunciation）、语调（intonation）、流利度（fluency）

### 2.4 智能笔记 / 摘要
- AI 自动生成每集摘要：关键观点、金句、实用表达
- 自动生成闪卡
- 笔记同步到网页端

### 2.5 生词本
- 保存查过的单词，含难度标注、发音、用法
- 集成 COCA 3000 词表
- 单词掌握标记

### 2.6 学习数据
- 每日听力时长记录
- 连续打卡追踪
- 成就解锁
- 学习曲线可视化图表

### 2.7 其他
- 离线下载
- PDF 导出（双语/纯英文转录稿）
- 暗黑模式（自动/手动切换）

## 3. UI/UX 设计

### 视觉风格
- **主色调**：深色背景（#0A0B0A, #0D0D0D），橙色强调色（#FF5301），白色文字
- **字体**：Inter，多字重层级
- **交互**：按钮 hover 缩放 110%，模糊效果，渐变叠加
- **整体感觉**：专业、沉浸、偏工具感

### 关键界面流程
1. **首页/发现页** → 按分类浏览播客 → 选择一集
2. **播放页**（核心页面）→ 音频播放 + 双语字幕滚动 + 点词交互
3. **跟读页** → 选择句子 → 录音 → AI 评分
4. **笔记/生词页** → 复习保存的内容
5. **数据页** → 查看学习进度

### 播放页的信息密度
这是 ListenLeap 最重要的页面。信息密度**很高**：
- 顶部：播客标题、集名
- 中间：双语字幕区（主体，占屏幕 60%+），当前句高亮
- 底部：播放控制条（播放/暂停、进度条、速度、循环、模式切换）
- 浮层：点词弹窗、AI 问答面板

## 5. 用户反馈

### 用户喜欢的
- 多难度分级，适合不同水平
- 适合每天碎片时间练习，保持语感
- 转录稿+跟读对考雅思很有帮助
- YouTube 导入 + 自动双语字幕非常方便
- 逐句分解帮助深度理解

### 用户抱怨的
- **免费版限制太多**——搜索和关注节目要付费
- **暗黑模式缺失/不完善**（已部分修复）
- **播放列表导航差**——无法跳转到当前播放的那一集，要在长列表中翻找
- **内容类型不够丰富**
- **没有免费试用期**

## 8. 对我们 demo 的启示

### ListenLeap 的本质
ListenLeap 本质上是一个**重学习工具**——它围绕"播客播放页"构建了一套完整的学习辅助系统（点词、跟读、笔记、生词本、AI 问答）。用户打开它的心理预期是"我要学习英语"。

### 如果我们要做"更轻松、更休闲的 ListenLeap"，核心差异应该是：

| 维度       | ListenLeap         | 我们的方向               |
| -------- | ------------------ | ------------------- |
| **用户心态** | "我要学英语"            | "我在刷内容，顺便听英语"       |
| **交互形态** | 播客播放器（横向信息密度高）     | 短视频 feed（纵向滑动，一屏一条） |
| **内容长度** | 整集播客（5–60 分钟）      | 单个片段（20–40 秒）       |
| **信息密度** | 高（字幕+控制条+点词+AI 面板） | 低（只有字幕和钩子标题）        |
| **用户操作** | 多（点词、切速、循环、跟读）     | 少（只有滑动）             |
| **内容发现** | 主动搜索/浏览分类          | 被动 feed，刷到什么听什么     |
| **学习压力** | 有（生词本、打卡、进度）       | 无（没有任何"学习"痕迹）       |
| **类比**   | 得到/喜马拉雅            | 抖音/TikTok           |
|          |                    |                     |
