# CEO 对话 Playbook:Flipod Pivot 决策

**日期**: 2026-04-15
**场景**: 1-on-1,目标是讨论 CEO 提出的"抛弃推荐 / 实时生成 AI Native"方向
**配套文档**:
- `2026-04-15-competitive-brief-ai-podcast-english-learning.md` (pivot 后的赛道)
- `2026-04-15-competitive-brief-current-flipod.md` (现状的赛道)

---

## 总原则

**不要打技术辩论赢道理输关系**。你能赢的最优结果不是"CEO 承认错了",是 **"CEO 自己选择了一个比他原始方向更好的版本"**。所以:

1. **先诊断,再开火**——不知道他真正的焦虑是什么,任何"武器"都打不到点上
2. **给 CEO 留台阶**——任何最终方案都让他能说"对,这就是我想要的"
3. **预设最差结果**——他坚持原案,你怎么继续干活,要提前想清楚

---

## Phase 0:开会前的自我准备(15 分钟)

### 想清楚 3 个问题

1. **你真正想要什么?**
   - (a) 完全否决 pivot,守住现有 Flipod
   - (b) 部分接受,用 Daily Brief 模块包装,主体守住
   - (c) 接受 pivot,但要求保留现有内容作为"premium / 编辑精选"
   - (d) 完全接受,只想避免最差的执行(乱花预算、抛弃用户)

   **建议:目标设 (b),容忍底线 (c),不要奢望 (a)。** 你是 PM 不是创始人,完全否决 CEO 方向的政治成本远高于收益。

2. **你不能丢的是什么?**(写下来,会议中守住这条线)
   - CEFR-J + 句级同步 + 词级标注的工程资产(切片或生成都用得上,不能扔)
   - 现有付费/活跃用户(不能因为 pivot 把他们清零)
   - 个人信用(你接的活要能交付,不能 6 个月后才发现做不出来)

3. **你能让步的是什么?**(提前想好,不要会议中临时决定)
   - 真人音频(可以让 TTS 替代)
   - 内容形态(可以从"播客片段"变"AI 生成 daily brief")
   - 推荐这个词(可以叫别的)
   - 22 条冷库(可以扔,反正小)

## Phase 1:开场——绝对不要做的事

### 错误开场示范

❌ "我做了竞品分析,我觉得这个方向有问题..."
→ 立刻进入辩论模式,CEO 防御姿态启动

❌ "Langua、Speak、NotebookLM 已经在做了,我们打不过..."
→ 听起来像是怯战 / 找借口

❌ "推荐系统其实就是生成系统,Meta 已经..."
→ 上来就"教育"CEO,关系成本极高

### 正确开场示范

✅ "我把现有 Flipod 和 AI 生成方向都做了一遍竞品分析,发现一些挺有意思的事,想跟你对一下。在我讲之前,我想先听听你——**你最想看到 Flipod 半年后是个什么样子?是用户量?是融资节点?是产品形态?**"

**关键**:把第一个问题让 CEO 回答,你听。这一步直接关乎你怎么用后面的"武器"。

---

## Phase 2:诊断 CEO 真实焦虑(对话中识别)

### 三种最可能的焦虑模式,识别信号 + 应对策略

| 焦虑模式 | 关键识别信号(听他说什么) | 你该用的武器 |
|---------|------------------------|-----------|
| **A. 融资叙事焦虑**:他认为推荐型产品融不到钱,资本市场只认 AI Native | 提到融资、估值、对标的是"AI 公司"而非产品功能;说"我们要讲一个 AI 故事" | 武器 1 + 武器 5 |
| **B. 对标刺激**:他看到某个 hot AI Native 产品,觉得 Flipod 不够"先进" | 多次提到具体公司名(Perplexity / Cursor / NotebookLM / 某创业公司);说"人家都已经..." | 武器 2 + 中间方案 |
| **C. 产品形态焦虑**:他觉得现在的 Flipod 太轻、太小、不够"性感" | 用"碎片化"、"不够大"、"22 条不像产品"这类词;关心"野心";说"我们要做点更大的" | 中间方案 + 武器 4 (单位经济) |
| **D. 不可移动的 vibe**:他就是认为这个方向对,不接受讨论 | 任何反驳都被打回;说"这就是方向,你执行就好";会议被打断 | 切换到 Phase 5 (合规执行 + 风险存档) |

### 诊断问题脚本(按顺序问,不要一次性问完)

1. "**半年后你最希望投资人/用户/团队怎么描述 Flipod?**" → 听他用什么形容词
2. "**最近你看到哪些产品让你觉得 Flipod 应该长那个样子?**" → 直接定位对标
3. "**你说推荐淘汰了——具体是什么让你下这个判断的?最近发生了什么?**" → 追溯触发事件
4. "**如果不动产品形态,我们还能在哪里把'AI Native'这个标签贴上去?**" → 测试他是要"AI Native 标签"还是"AI Native 产品"

如果他绕过具体回答,只用抽象词("就是不够 AI Native"、"方向就是这样"),那大概率是 **Mode D** 或 **Mode C**——问题不在技术,是情绪/直觉。

## Phase 3:武器库——按诊断匹配

> ⚠️ 武器只在诊断完之后用。诊断错 → 武器打偏 → 关系受损。

### 武器 1:推荐 vs 生成是伪二分法(对应焦虑 A:融资叙事)

**核心说法**:
> "我特别同意 AI Native 是方向。但我最近读到的东西让我想跟你对一个细节——前沿的推荐系统这两年正在变成生成式系统。Meta 2024 年的 'Actions Speak Louder than Words' 论文直接把推荐改成了 trillion-parameter sequential transducer,Pinterest 的 PinFormer、字节的 HLLM 都是同样思路。底层都是 p(next | context),区别只是 token 是单词还是 item。所以问题可能不是'要不要推荐',而是'推荐这层要不要也用 generative model 重写'。"

**这个说法的好处**:
- 不直接否定他,反而把他的方向往前推了一层
- 给他一个新的、更前沿的"AI Native"叙事(资本爱听)
- 你不是反对者,你是"补全他直觉的人"

**风险**:
- CEO 如果不熟悉机器学习背景,可能只听到"你在掉书袋"
- 必须配一个"转化成产品语言"的版本(见武器 5)

---

### 武器 2:让他举反例(对应焦虑 B:对标刺激)

**核心说法**:
> "我想跟你确认一下定义。**纯实时生成、零推荐的 AI Native 消费产品**,你脑子里有哪几个标杆?我去拆解一下他们怎么做的,我们就照那个学。"

**预期**:他会列出 ChatGPT / Perplexity / Cursor / NotebookLM / Character.ai 等。

**然后你**(温和地):
> "有意思——这几个我都用过。我们一个个看:ChatGPT 有 history 和 GPT Store ranking,Perplexity 有 Discover feed,Cursor 的 tab autocomplete 本质就是对你下一步代码的推荐,Character.ai 100% 是推荐驱动。**好像没有一个真的'零推荐'。** 也许我们要的不是'抛弃推荐',而是'把推荐做得更生成、更智能'?"

**这个说法的好处**:
- 用他自己的对标产品反推,他无法说"那些产品做得不对"
- 不是你说他错,是数据让他自己得出结论

**风险**:
- 如果 CEO 没用过这些产品就开口判断,这会让他下不来台。**用之前判断他对这些产品的熟悉度。**
- 一定要用"我们" / "我"的视角,不要"你"

---

### 武器 3:用户认知负担(对应焦虑 B 或 C)

**核心说法**:
> "我有个担心想跟你说——**实时生成隐含一个假设:用户知道自己要什么。** 在 ChatGPT 这种工具场景成立(用户带着明确问题来)。但在我们这种内容消费场景,大多数学习者不知道自己英语水平、不知道哪个话题适合自己、不知道哪条新闻对自己有用。这个信息缺口必须有人补——要么编辑团队补(不 scalable),要么算法补(就是推荐)。**Artifact 死的就是这个原因——它假设用户知道想读什么,结果用户打开 app 不知道输入啥就关掉了。** 我们要避免这个陷阱。"

**这个说法的好处**:
- 引用一个真死掉的产品(Artifact)做反面教材,情感冲击力强
- 把"推荐"重新框定为"补认知缺口的必要机制",而不是"老旧技术"

**风险**:
- 如果 CEO 没听过 Artifact,需要 30 秒铺垫(BuzzFeed 创始人做的 AI news app,2024 年关停)

---

### 武器 4:单位经济(对应焦虑 C 或 A)

**核心说法**:
> "我跑了一下数。如果是真正的'每个用户打开都实时生成',pipeline = 抓取 + LLM 改写 + TTS + 对齐,单次成本几毛到几块,延迟分钟级。一个 DAU 每天打开 3 次 = 一个用户每月成本 ~$10-30。**ARPU 撑不住,CAC/LTV 算不通。** 所以工程上'实时生成'几乎一定是'预生成 + 个性化召回'的混合体——而个性化召回就是推荐。**这不是产品理念问题,是物理约束。** 我建议我们一开始就承认这点,把推荐这一层做得 AI Native,而不是先扔了再发现要捡回来。"

**这个说法的好处**:
- 数学是 CEO 最难反驳的——他可以否定你的判断,不能否定单位经济
- 给他一个台阶:"不是不做,是物理上必须这样做"

**风险**:
- 你的数字必须站得住脚。**会议前一天再核一遍**(TTS 单价 $0.30/1k chars × 平均 clip 长度 × 用户日均打开次数)。出现一个明显错的数,整个论点崩

---

### 武器 5:重命名(对应焦虑 A,辅助任何场景)

**核心策略**:
不说"推荐算法",说 **"个性化生成的 context"** 或 **"用户意图建模"** 或 **"AI Native 的内容编排层"**。

**示例对话**:
> CEO: "我不要推荐。"
> 你: "完全同意。我们不做推荐——我们做'**AI Native 的意图理解层**',它根据用户兴趣 + 当前水平 + 时段,**生成**一个个性化 feed。这跟传统协同过滤完全不是一回事。"

实际产品可能 60% 是推荐,40% 是生成,但语言外壳是 100% AI Native。

**这个说法的好处**:
- 政治成本最低
- CEO 反对的是"推荐"这个符号,不一定是这件事
- 你和他都能赢——他赢了"AI Native"的叙事,你赢了"用对了技术"

**风险**:
- 团队内部需要清晰沟通"对外叫 X 对内做 Y",否则工程师困惑
- 长期可能形成"言行不一"的产品文化,慎用

---

## Phase 4:中间方案(任何焦虑都可以收尾用)

如果武器都用过 CEO 还是不让步,**最后一招是把 pivot 重新定义,不是否决它**:

**提案脚本**:
> "我接受方向。**Flipod 升级版 = AI Native Daily Brief(主推 + AI 故事载体) + Editors' Picks(精选真实播客片段,作为 premium tier)**。这样:
>
> - **AI Native 标签** ✓ 你要的故事讲完
> - **抓取真实信息 + AI 转写** ✓ 实时性 + 个性化都拿到
> - **现有内容资产** 不浪费,变成 premium 的差异化
> - **现有付费用户** 不流失,还可能因为新功能升级订阅
>
> 这样我们既能讲 AI Native 给资本市场,又有真实内容护住学习者认知。**两条腿走路,不是赌一边。**"

**为什么这个收尾有用**:
- CEO 拿到他想要的(AI Native 主品牌)
- 你保住你想要的(现有 Flipod 资产、CEFR-J 工程、付费用户)
- 没有人输

**预期 CEO 反应**:
- 大概率接受(60%+),因为他只要"AI Native"故事,不是真要烧掉现有产品
- 中等概率说"那真实播客那部分小一点"(可接受,你保住核心)
- 小概率坚持"全 pivot"——这时候你进入 Phase 5

## Phase 5:守不住怎么办(Mode D 或谈崩)

### 如果他完全不让步:**别在会议中继续 push**

- 关系成本高,赢了道理输了关系
- 情感层面已经堵塞,逻辑过不去
- 延后:"我理解。我回去想想怎么最好地落地这个方向,3 天内给你一份执行 plan,我们再对一次。"

### 会后 24 小时内你必须做的 3 件事

1. **写一份 risk memo**(给自己,不给 CEO)
   - 列出你预见的 3-5 个风险(用户流失、Langua 竞争、单位经济、合规、TTS 听感)
   - 每个风险标 trigger condition(发生什么算这个风险被验证)
   - 存档,3-6 个月后回头看。如果 trigger 触发,你有数据 push 课程修正,而不是"早说了"

2. **给现有付费/活跃用户发一封"产品演进"沟通**
   - 不预告 pivot,但收集他们对现有产品最在乎什么
   - 如果回答里"真实播客 + 真人声音"占比高,这是关键 signal,会议中给 CEO 用

3. **跟工程团队对一下 carryover 资产**
   - CEFR-J、句级同步、翻译、词级标注 → 哪些能直接复用到 AI 生成 pipeline
   - 不让 pivot 等于"从零开始",这能省 40-60% 工作量

---

## 关键守则:会议中的几个红线

1. **不要在 CEO 面前说"竞品已经做了"** → 听起来像怯战
   - 改说:"市场上已有玩家给我们提供了参考路径"
2. **不要用"但是" / "不过"** → 触发防御
   - 改用 "and" 思维:"是的,而且我注意到..."
3. **不要超过 2 次 push back** → 第三次开始就是关系账户透支
   - 第一次 push back:数据
   - 第二次 push back:中间方案
   - 第三次:停手,延后再谈
4. **不要在群里 / 邮件里继续争论** → 文字记录把双方都钉死,以后退不出
   - 重要争论必须当面或电话
5. **不要让团队听到你和 CEO 争论** → 团队会站队,无论谁赢都伤组织

## Appendix:几句"金句"备用

会议中可能用得上:

- "我同意我们要 AI Native——我想跟你对的是**怎么定义'AI Native'**最有杀伤力。"
- "前沿推荐系统这两年正在变成生成式系统,不是要不要推荐,是怎么用 LLM 重写推荐。"
- "你说的我都收到。我有一个担心想分享:[Artifact 案例 / 单位经济数字]。我想在拍板前确认我们有 plan B 应对它。"
- "如果方向定了,我接。我想跟你对一下保留哪些现有资产能让 pivot 更快——CEFR、句级同步、翻译这套不复用就太可惜了。"
- "我先回去想 3 天,把执行 plan 写细。如果中间发现问题我们再对。"

## 决策后的事(不管哪条路)

| 结果 | 你立刻该做的 |
|------|------------|
| 完全守住现状 | 加 mobile app + SRS,启动用户研究,跟 CEO 季度对一次"AI Native 怎么补" |
| 中间方案(Daily Brief + Editors' Picks) | 起 Daily Brief PRD v0.1,设计两个产品的关系,守住 CEFR 工程资产 |
