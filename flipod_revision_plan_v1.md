## 背景

**当前症状**
- UI 视觉密度高、选项多、文字多
- 五阶段心智模型在第一屏没建立 → 用户看不出产品在干嘛
- 收到两类内部反馈：CEO 说"UI 太复杂"、同事说"切片太难"

**对内部反馈的判定**
- 这两人都不是目标用户，都是 lousy PM。反馈是 stakeholder 政治不是用户研究
- 字面采纳会让产品偏离定位（UI 简到 CEO 满意 + 难度降到同事满意 = 一个面向初学者的简单 app）
- 但反馈触发的两个**根因洞察**是真的，需要解决：
  - 首屏 30 秒不能 grok 产品（不是按钮问题）
  - 用户面对难内容时归因到"我笨"而非"内容难"（不是难度问题）

**不变量（任何改版决策的边界）**
- 目标用户：中级听力 + 愿意听播客
- 定位：**刷感外壳 + 教学专业引擎 + 陪伴型内部基调**
- 内容客观难度不降
- 五阶段练习结构作为产品脊柱
- "陪伴"不进外部文案

## 改什么（4 个具体动作）

### 1. 信息架构收敛 —— 砍纯听 tab

- **砍**：纯听 tab
- **改**：Phase 0 = 纯听（extensive listening，建立语流感）
- **为什么**：两个并行入口（"刷"和"练"）让用户在第一屏就要选"今天想轻松还是想认真"，这本身是认知负担。收敛成单一脊柱 → 用户永远只面对一个流程，Phase 0 天然承担"轻松"模式
- **副作用**：解决一部分"界面复杂"的真问题

### 2. 引入音波智能体 —— 严格限定两个时机

**时机 A：首次打开（onboarding 30 秒闭环）**

```
音波出现 → "想听 30 秒试试吗？"
  → 一键开始 → 播放一段精选 clip
  → 听完 → "懂了多少？我帮你拆一遍"
  → 自然进入 phase 1（拆一两个关键词/连读）
  → 完成 → "看，刚才没全懂的现在懂了 —— 这就是 Flipod 在做的事"
```

胜利标志：30 秒内用户体验完整价值闭环（听 → 没全懂 → 被帮助 → 进步了）

**时机 B：Clip 之间（framing + 推荐）**

每段 clip 之前一句话提示：
- "下一段语速比较快，目标是抓住主旨"
- "下一段试着跟读节奏"
- "因为你刚才那段连读卡了一下，这段连读模式类似"

询问轻松/挑战："想轻松一点还是难一点？"

**绝对不出场的时机**：clip 内部所有 phase
- 教学专业引擎承担教学，对话会稀释结构
- 音波形象在 clip 之内退场，让结构化 UI 接管

### 3. 首屏重构 —— 让 30 秒 grok

**当前问题**：用户打开 app 看到的是"五阶段练习的入口"，一个还没建立动机的人不可能 get 到这意味着什么

**新首屏**：
- 音波形象 + 一句话欢迎
- 一个核心 CTA（开始 30 秒体验）
- **不要**：分类菜单、难度选择、tab 切换
- 用户走完 30 秒闭环之后，才渐进暴露完整产品结构（progressive disclosure）

### 4. Microcopy 系统 —— 陪伴型基调进每一句

每个用户接触点按"温和、不评判、有方法论"重写：

| 场景 | ❌ 业余写法 | ✅ Flipod 写法 |
|------|------------|--------------|
| 答错时 | "错了" / "再试一次" | "这个连读容易卡 —— 听一遍慢速版？" |
| 长期未访问 | "x 天没来打卡了" | "上次卡的那段连读，要不要再试试？" |
| Phase 过渡 | "进入下一阶段" | "刚才你摸到了节奏，现在我们抓细节" |
| 完成片段 | "+10 经验值！" | "这段你抓到了 70%，比上一条进步了" |

## 视觉 / 参照系

**学**：
- Headspace 的角色温度 —— 温柔、不施压、可信赖
- Duolingo 的引导式 onboarding —— 首次打开就让用户做事而非看说明
- TikTok 的滑动节奏 —— 流畅、低承诺、可中断

**不学**（明确反参照）：
- Replika 的情感陪聊 —— 避免被误读为"AI 男/女朋友"
- 每日英语听力/可可英语的 menu-driven 界面 —— 那是不刷的旧形态
- 学习打卡类产品的 KPI 化压力 —— 与陪伴基调冲突

## 不做清单（明确边界）

- ❌ 不降低 clip 内容客观难度
- ❌ 不为初学者设计任何功能
- ❌ 不在 clip 内部使用对话 UI
- ❌ 不在外部文案 / AppStore 描述里出现"陪伴"二字
- ❌ 不立刻完善后端推荐算法（等真实用户信号）
- ❌ 不为响应 CEO 的具体 UI 简化建议而砍 phase 数量
- ❌ 不让智能体替代有结构化语义的按钮（播放/暂停/速度等）
- ❌ 不在没有真实用户数据前响应 stakeholder 的具体产品建议

## 自检清单（做之前 + 做之后都过一遍）

1. **首屏测试**：一个完全不知道 Flipod 是什么的人，30 秒内能 grok 产品价值吗？
2. **智能体台词**：这句话是在帮 grok / framing / 归因管理？还是只是替代了一个按钮？后者要删
3. **专业引擎**：五阶段每一阶段对应的听力训练目标，能用一句话讲清吗？（extensive listening / intensive listening / bottom-up / top-down / shadowing 等）
4. **定位边界**：这个决策偏离"中级用户定位"了吗？
5. **基调一致性**：这条 microcopy 是温和不评判？还是滑向了督促/打卡/KPI？

## 验证计划（最重要 —— 不做这步前面都白搭）

**优先级**：找真实用户 > 完善智能体设计 > 回应 stakeholder 反馈

**具体步骤**：

1. **招募 5-10 个真实目标用户**
   - 画像：中级听力（雅思 6-6.5 / CET6 通过 / 能听懂 60-80% 真实播客）+ 愿意听播客
   - 来源：身边的人、播客听众社群、小红书相关话题、播客类播主的粉丝群
   - 不要：英语很差的人（不是目标用户，反馈会误导）

2. **测试材料：MVP demo**
   - 不需要全功能 —— Phase 0 + Phase 1 + onboarding 闭环 + 1-3 条 clip 就够
   - 重点测的是产品心智，不是功能完备

3. **观察三个时间维度的反应**
   - 30 秒后：能 grok 产品在干嘛吗？
   - 3 分钟后：完成第一个完整 clip 后，是放松还是挫败？
   - 3 天后：还会主动再打开吗？

4. **每周复盘**
   - 真实用户信号 vs lousy PM 反馈的偏差
   - 用真实信号校准产品方向，不用 PM 反馈

**搁置**：在拿到至少 5 个目标用户的真实使用数据之前
- 不响应 CEO 的具体 UI 建议（用"给他能 demo 的 30 秒叙事"满足他的真实 JTBD）
- 不响应同事的具体难度建议
- 不做大规模重构

## 一句话总结

**通过引入"音波智能体"承担首印象主演 + 难度心理框架师两个职能，砍纯听 tab 收敛信息架构，让产品在 30 秒内 grok-able 且帮助用户把"听不懂"归因到内容而非自己 —— 在不降客观难度、不偏离中级用户定位的前提下解决"UI 复杂"和"切片难"两个反馈背后的真根因。**
